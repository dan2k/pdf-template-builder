# คู่มือการใช้งานระบบสร้างเทมเพลต PDF (User Manual)

ยินดีต้อนรับสู่ระบบสร้าง PDF Template แบบครบวงจร! ระบบที่จะช่วยให้การสร้างเอกสาร PDF เช่น ใบเสร็จรับเงิน, ใบแจ้งหนี้ หรือเอกสารราชการ เป็นเรื่องง่ายและรวดเร็ว โดยไม่จำเป็นต้องเขียนโปรแกรม

---

## 1. ภาพรวมระบบ (System Overview)

ระบบนี้คือเครื่องมือออกแบบและสร้างเอกสาร PDF ที่ทำงานบนเว็บเบราว์เซอร์ โดยใช้หลักการ **"ลากและวาง" (Drag & Drop)** เพื่อจัดวางองค์ประกอบต่างๆ เช่น ข้อความ รูปภาพ ตาราง และรูปทรง ลงบนหน้ากระดาษอย่างอิสระ

**ระบบนี้ช่วยแก้ปัญหาอะไร?**
- **ลดเวลาในการออกแบบเอกสารที่ซับซ้อน**: แก้ไขรูปแบบเอกสารได้ทันทีผ่านหน้าเว็บ
- **แยกหน้าที่ระหว่างคนออกแบบ (Designer) และนักพัฒนา (Developer)**:
  - **Designer/Admin**: สามารถดูแลความสวยงาม จัดวางโลโก้ หรือเปลี่ยนฟอนต์ได้ด้วยตัวเองโดยไม่ต้องรอโปรแกรมเมอร์มาแก้ไขโค้ด
  - **Developer**: มีหน้าที่เพียงแค่ส่งข้อมูลชุดเดิมผ่าน API ระบบจะทำการวาดเอกสารตามเทมเพลตใหม่ให้ทันที
  - **ข้อดี**: ลดคอขวด (Bottleneck) ในการทำงาน, ทีมกราฟิกสามารถแก้ไขงานได้รวดเร็วขึ้น และนักพัฒนามีเวลาไปโฟกัสกับ Logic ของระบบหลักได้มากขึ้น
- **รองรับ AI Assistant หลายค่าย**: จัดการทุกอย่างง่ายๆ ผ่าน Admin Panel (Support: Gemini, ChatGPT, Claude, OpenRouter, Local LLM) พร้อมระบบแปลงไฟล์รูปภาพ **SVG** อัตโนมัติเพื่อให้แสดงผลใน PDF ได้ทุกเครื่อง
- **รองรับข้อมูลที่เปลี่ยนแปลงได้ (Dynamic Data)**: ดึงข้อมูลชื่อลูกค้าหรือรายการสินค้าจากระบบอื่นมาใส่ใน PDF ได้โดยอัตโนมัติ

---

## 2. คุณสมบัติหลัก (Key Features)

- **Template Editor**: ออกแบบ PDF ได้เหมือนใช้โปรแกรมวาดภาพ รองรับการหมุน, ย่อ-ขยาย และจัดลำดับชั้นขององค์ประกอบ
- **AI Assistant**: สั่งงานด้วย AI ผ่านข้อความภาษาไทยหรืออังกฤษ เพื่อให้ระบบช่วยวางองค์ประกอบหรือออกแบบเบื้องต้นให้ทันที
- **Dynamic Elements**: ใส่ค่าตัวแปรในรูปแบบ `{{ตัวแปร}}` เพื่อเปลี่ยนข้อมูลใน PDF ตามข้อมูลที่ส่งเข้ามา
- **Multi-page Support**: รองรับการสร้างเอกสารหลายหน้า และการจัดการ Header/Footer ที่ปรากฏทุกหน้า
- **API Integration**: มีระบบ API Key สำหรับให้นักพัฒนาเชื่อมต่อระบบอื่นเพื่อดาวน์โหลด PDF ได้โดยอัตโนมัติ

## 3. ระดับผู้ใช้งาน (User Levels)

เพื่อให้การจัดการเอกสารมีความปลอดภัยและเป็นระเบียบ ระบบจึงแบ่งสิทธิ์การเข้าถึงออกเป็น 3 ระดับ ดังนี้:

### 3.1 ผู้ดูแลระบบ (Admin)
- **สิทธิ์สูงสุด**: สามารถจัดการผู้ใช้งานทุกคนในระบบ (เพิ่ม/ลบ/รีเซ็ตรหัสผ่าน) และจัดการหน่วยงาน (Department)
- **การจัดการเอกสาร**: สามารถดู แก้ไข หรือลบเทมเพลตของทุกคนได้ และสามารถคัดลอก (Duplicate) เทมเพลตข้ามผู้ใช้ได้
- **การจัดการหมวดหมู่**: สามารถสร้างหมวดหมู่ส่วนกลางและจัดการหมวดหมู่ของทุกคนได้

### 3.2 ผู้ใช้งานทั่วไป (User)
- **การจัดการเอกสาร**: สร้างและแก้ไขเทมเพลตของตัวเองได้เท่านั้น
- **สิทธิ์การแชร์**: สามารถตั้งค่าเทมเพลตเป็น "Public" เพื่อให้ผู้อื่นเห็น หรืออนุญาตให้ผู้อื่นคัดลอก (Allow Copy) ได้
- **การจัดการส่วนตัว**: จัดการหมวดหมู่ (Category) และแท็ก (Tag) ของตัวเองได้ รวมถึงการสร้าง API Key สำหรับเทมเพลตตัวเอง

### 3.3 ผู้เยี่ยมชม (Guest)
- **การเข้าถึง**: สามารถเข้าดูเทมเพลตที่ตั้งค่าเป็น **"Public"** (สาธารณะ) ได้เท่านั้น
- **ฟังก์ชันที่ใช้งานได้**: สามารถใช้งานฟังก์ชัน **Generate PDF** (กรอกข้อมูลและดาวน์โหลด PDF) จากเทมเพลตสาธารณะได้
- **ข้อจำกัด**: ไม่สามารถสร้าง/แก้ไขเทมเพลต ไม่สามารถจัดการหมวดหมู่ และไม่สามารถเข้าถึงข้อมูลส่วนตัวของผู้อื่นได้

## 4. วิธีใช้งาน Dashboard (หน้าแรก)

หน้า Dashboard คือจุดศูนย์กลางในการจัดการเอกสารทั้งหมดของคุณ ประกอบด้วยส่วนสำคัญดังนี้:

- **Sidebar (แถบด้านข้าง)**: 
  - **All Templates**: แสดงเทมเพลตทั้งหมดที่คุณมีสิทธิ์เข้าถึง
  - **Departments & Users (เฉพาะ Admin)**: กรองเทมเพลตตามหน่วยงานหรือตามรายชื่อผู้ใช้
  - **Categories**: กรองเทมเพลตตามหมวดหมู่ที่คุณสร้างขึ้น (รองรับการทำหมวดหมู่ย่อย)
- **Top Bar (แถบด้านบน)**:
  - **Search**: ค้นหาเทมเพลตอย่างรวดเร็วด้วยชื่อ, รายละเอียด หรือแท็ก
  - **Tags**: คลิกที่แท็กต่างๆ เพื่อกรองเฉพาะเทมเพลตที่เกี่ยวข้อง
  - **Create New Template**: ปุ่มสำหรับเริ่มสร้างเทมเพลตใหม่
- **Template Card**: 
  - แสดงพรีวิวของเอกสาร, ชื่อ, และข้อมูลหมวดหมู่
  - มีปุ่มลัดสำหรับ **Edit** (แก้ไข), **Generate** (พิมพ์เอกสาร), **Duplicate** (คัดลอก) และ **Settings** (ตั้งค่า)

## 5. วิธีใช้งานทีละขั้นตอน (Step-by-Step Guide)

### 5.1 การสร้างและแก้ไขเทมเพลต
1. ในหน้า Dashboard, คลิกปุ่ม **"Create New Template"**
2. ที่แถบเครื่องมือด้านบน เลือกเพิ่มองค์ประกอบต่างๆ เช่น **Text**, **Image**, **Table**, หรือ **Shapes**
3. คลิกที่องค์ประกอบเพื่อปรับค่าในแถบ **Properties** ด้านขวา (เช่น สี, ฟอนต์, ขนาด)
4. **Data Binding**: พิมพ์ชื่อตัวแปรในข้อความ เช่น `{{customerName}}` เพื่อให้ระบบแทนที่ด้วยข้อมูลจริง (ดูรายละเอียดการเพิ่มตัวแปรในข้อ 5.2)

### 5.2 การจัดการตัวแปร (Variable Management)
ก่อนที่จะนำตัวแปรไปใช้งานในเอกสาร คุณต้องกำหนดชื่อและประเภทของตัวแปรในระบบก่อน ดังนี้:
1. คลิกที่ **พื้นที่ว่างบน Canvas** (เพื่อให้แถบ Properties ด้านขวาแสดงข้อมูลของหน้ากระดาษ)
2. ที่แถบด้านขวา คุณจะพบส่วน **"Variables"**
3. คลิกปุ่ม **"+ Add"** เพื่อเพิ่มตัวแปรใหม่
4. กำหนดรายละเอียดของตัวแปร:
   - **Name**: ชื่อตัวแปร (ต้องตรงกับชื่อที่ใช้ใน JSON และในเครื่องหมาย `{{ }}`)
   - **Type**: ประเภทข้อมูล (String, Number, Date หรือ Array สำหรับตาราง)
   - **Default Value**: ค่าเริ่มต้นที่จะแสดงหากไม่มีการส่งข้อมูลเข้ามา
5. กดปุ่ม **Save (`Ctrl + S`)** เพื่อบันทึกการตั้งค่าตัวแปร

### 5.3 การใช้ AI Assistant
คลิกปุ่ม **"AI"** (รูปไม้กายสิทธิ์) กรอกคำสั่ง เช่น *"สร้างตารางรายการสินค้า 5 คอลัมน์ และเปลี่ยนสีหัวตารางเป็นสีเขียว"* จากนั้นกด **Generate** โดยระบบจะเลือกใช้ Provider และ Model ตามที่ Admin กำหนดไว้ในส่วนกลางครับ

### 5.4 การตั้งค่าหัวกระดาษ/ท้ายกระดาษ และเลขหน้า (H/F & Page Number)
หากคุณต้องการให้เอกสารมีข้อมูลปรากฏในทุกหน้า (เช่น โลโก้บริษัท หรือเลขหน้า) สามารถตั้งค่าได้ที่แถบ **"H/F"** ในด้านขวา:
1. **หัวกระดาษ/ท้ายกระดาษ (Header/Footer)**:
   - ติ๊กถูกที่ **"Enabled"** เพื่อเปิดใช้งานในส่วนที่ต้องการ
   - **ใช้กับทุกหน้า (Apply to all pages)**: เปิดสวิตช์นี้เพื่อให้การแก้ไขที่จุดเดียว มีผลกับทุกหน้าในเทมเพลตโดยอัตโนมัติ
   - คุณสามารถลากวาง Text, Image หรือ Shape ลงในพื้นที่ Header/Footer ได้โดยเฉพาะ
2. **การใส่เลขหน้า (Page Number)**:
   - ไปที่แท็บย่อย **"Page No."** ในแถบ H/F
   - เลือกรูปแบบที่ต้องการ (เช่น `หน้า {page} / {total}`) ระบบจะแทนที่ `{page}` ด้วยเลขหน้าปัจจุบัน และ `{total}` ด้วยจำนวนหน้าทั้งหมด
   - ปรับตำแหน่ง (บน/ล่าง, ซ้าย/กลาง/ขวา) และสไตล์ (ฟอนต์, ขนาด, สี) ได้ตามต้องการ

### 5.5 การจัดการข้อมูลเทมเพลต (Info Tab)
แถบ **"Info"** ใช้สำหรับจัดการข้อมูลทั่วไปและสิทธิ์ของเทมเพลต:
- **Description**: ใส่คำอธิบายสั้นๆ เกี่ยวกับเทมเพลตเพื่อให้ค้นหาได้ง่ายขึ้น
- **Category & Tags**: จัดหมู่เอกสาร (เช่น "ใบเสร็จ", "ฝ่ายบัญชี") และเพิ่มแท็ก (Tags) เพื่อความสะดวกรวดเร็วในการกรองข้อมูลบน Dashboard
- **Visibility**: 
  - **Private**: เห็นและใช้งานได้เฉพาะคุณคนเดียว
  - **Public**: ให้ผู้ใช้งานคนอื่นเข้ามาดูและ Generate PDF ได้
- **Allow Copy**: หากเปิดใช้งาน ผู้อื่นจะสามารถคัดลอก (Duplicate) เทมเพลตของคุณไปเป็นของตัวเองได้

### 5.6 การจัดการ API Key (API Key Management)
API Key เปรียบเสมือนกุญแจที่ใช้สำหรับการสั่งพิมพ์เอกสารจากระบบภายนอก โดยไม่ต้องมีการล็อคอินผ่านหน้าเว็บ **ใครมีสิทธิ์จัดการ?** เฉพาะเจ้าของเทมเพลตและ Admin เท่านั้นที่สามารถสร้างหรือลบ Key ได้

**ขั้นตอนการสร้าง API Key:**
1. ในแถบ **"Info"** เลื่อนลงมาด้านล่างสุดจะพบส่วน **"API Keys"**
2. คลิกปุ่ม **"Generate Key"**
3. (ระบุชื่อถ้าต้องการ) เช่น "Server สำนักงานใหญ่" แล้วกด **Generate**
4. **ข้อควรระวัง**: ระบบจะแสดงรหัส Key ให้ดู **เพียงครั้งเดียวเท่านั้น** โปรดคัดลอกและเก็บไว้ในที่ปลอดภัยทันที หากทำหายจะไม่สามารถเรียกดูใหม่ได้ (ต้องลบแล้วสร้างใหม่)

**การเพิกถอนสิทธิ์ (Revoke):**
- คุณสามารถยกเลิกการใช้งาน Key ได้ตลอดเวลาโดยกดปุ่ม **ถังขยะ (Revoke)**
- เมื่อยกเลิกแล้ว ระบบภายนอกที่ใช้ Key นั้นจะ **ไม่สามารถ** สั่งพิมพ์เอกสารได้ทันที

### 5.7 การพิมพ์เอกสาร (Generate PDF)
เมื่อออกแบบเสร็จแล้ว ให้ไปที่หน้า **Generate PDF** (จากปุ่มลัดบน Card หรือใน Editor):
1. **Input Data**: ใส่ข้อมูลในรูปแบบ JSON (หากไม่ชำนาญ ให้กดปุ่ม **"Sample"** เพื่อโหลดตัวอย่างข้อมูล)
2. **Preview**: กดปุ่ม **Preview** เพื่อดูผลลัพธ์แบบ PDF ทันทีที่หน้าจอ
3. **Download**: เมื่อพอใจแล้ว กดปุ่ม **Download PDF** เพื่อบันทึกไฟล์ลงเครื่อง
4. **API Usage**: สำหรับนักพัฒนา สามารถคัดลอก **cURL Example** ไปใช้งานในระบบอื่นได้ทันที (ดูรายละเอียดคู่มือ API ในข้อ 7)

## 6. คำถามที่พบบ่อย (FAQ) และวิธีแก้ปัญหาเบื้องต้น

**Q: ต้องการให้ตารางแสดงผลซ้ำในหน้าถัดไปเมื่อข้อมูลมีจำนวนมาก ต้องทำอย่างไร?**
- **A**: คลิกที่ตาราง แล้วไปที่แถบ Properties ติ๊กถูกที่ช่อง **"Repeat header on new page"**

**Q: รูปภาพที่อัปโหลดไม่แสดงผลใน PDF?**
- **A**: ตรวจสอบขนาดไฟล์ภาพ (ไม่ควรเกิน 2MB) และตรวจสอบให้แน่ใจว่าได้กด Save หลังจากอัปโหลดรูปภาพแล้ว

**Q: ทำไมบันทึกเทมเพลตแล้วข้อมูลไม่เปลี่ยน?**
- **A**: ตรวจสอบว่ามีสถานะ "Unsaved" ปรากฏอยู่หรือไม่ แนะนำให้กดปุ่ม **Save** หรือใช้คีย์ลัด `Ctrl + S` และรอจนขึ้นสถานะ "Saved"

## 7. คู่มือการใช้งาน API (External API Guide)

หัวข้อนี้สำหรับนักพัฒนาที่ต้องการเชื่อมต่อระบบภายนอกเพื่อสั่ง Generate PDF โดยอัตโนมัติ

### 7.1 การยืนยันตัวตน (Authentication)
การเรียกใช้งาน API ต้องส่ง **API Key** ของเทมเพลตนั้นๆ ผ่าน HTTP Header ชื่อ `X-API-Key`
> [!IMPORTANT]
> API Key จะขึ้นต้นด้วย `sk_` เสมอ (เช่น `sk_123.abc...`) สามารถสร้างได้จากหน้า **Settings** ของแต่ละเทมเพลต

### 7.2 เลือกรูปแบบผลลัพธ์ (Response Type)
คุณสามารถกำหนดรูปแบบผลลัพธ์ได้ 2 แบบ ผ่าน Parameter `responseType` ใน JSON Body:

| รูปแบบ | สิ่งที่จะได้รับ | เมื่อไหร่ที่ควรใช้ |
| :--- | :--- | :--- |
| **PDF Binary** (ค่าเริ่มต้น) | ไฟล์ PDF โดยตรง | เมื่อต้องการดาวน์โหลดไฟล์ทันที หรือใช้ `curl` ทดสอบ |
| **Base64 JSON** | ข้อความ JSON: `{ "base64": "..." }` | สำหรับนักพัฒนาที่ต้องการนำข้อมูลไปประมวลผลต่อในโค้ด |

### 7.3 ตัวอย่างการใช้งานด้วย cURL

**แบบที่ 1: รับข้อมูลเป็นไฟล์ PDF (แนะนำสำหรับการทดสอบ)**
```bash
curl -X POST "http://localhost:3000/templates/YOUR_TEMPLATE_ID/generate" \
     -H "X-API-Key: YOUR_API_KEY" \
     -H "Content-Type: application/json" \
     -d '{
           "name": "John Doe"
         }' \
     --output document.pdf
```

**แบบที่ 2: รับข้อมูลเป็น JSON Base64 (สำหรับนักพัฒนา)**
```bash
curl -X POST "http://localhost:3000/templates/YOUR_TEMPLATE_ID/generate" \
     -H "X-API-Key: YOUR_API_KEY" \
     -H "Content-Type: application/json" \
     -d '{
           "responseType": "base64",
           "name": "John Doe"
         }'
```
